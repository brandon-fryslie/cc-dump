Scope: Adding User and Assistant message collapse/expand filters to cc-dump TUI

## Current State

### Filter System Architecture
The TUI has a well-established filter system with the following components:

1. **Filter state**: `CcDumpApp` holds reactive booleans (`show_headers`, `show_tools`, `show_system`, `show_expand`, `show_metadata`) exposed via `active_filters` property as a dict.
2. **Key bindings**: Each filter has a single-key toggle (h, t, s, e, m) defined in `BINDINGS`.
3. **Reactive watchers**: Changes to filter state trigger `_rerender_if_mounted()` which calls `conv.rerender(filters)`.
4. **Block-level filtering**: `rendering.py` has a `_BLOCK_REGISTRY` mapping block type names to `(renderer_fn, filter_key)`. Renderers return `None` to hide a block.
5. **Filter relevance**: `TurnData.compute_relevant_keys()` determines which filter keys affect a turn, enabling early-exit in `re_render()`.
6. **Tool collapse**: When tools filter is off, consecutive `ToolUseBlock`s are collapsed into a summary line via `_make_tool_use_summary()` in `render_blocks()`.

### Block Types for Messages
- `RoleBlock`: Renders role labels (USER, ASSISTANT, SYSTEM). Has `role` field. Filter key: `"system"` (only hides system roles).
- `TextContentBlock`: Renders plain text content. Has `text` and `indent` fields. Filter key: `None` (always visible).

### Key Observation: No Per-Role Filtering Exists
Currently, User and Assistant message content (`TextContentBlock`) is ALWAYS shown in full. The only content collapsing mechanism is:
- `TrackedContentBlock` (system prompt diffs) controlled by `"expand"` filter
- `TurnBudgetBlock` controlled by `"expand"` filter
- Tool blocks controlled by `"tools"` filter

### Rendering Pipeline
1. `formatting.py` produces blocks including `RoleBlock` + `TextContentBlock` sequences
2. `rendering.py:render_blocks()` iterates blocks, calls per-block renderers, builds `(block_index, Text)` pairs
3. `rendering.py:render_turn_to_strips()` converts Text objects to Strip objects for Line API
4. `widget_factory.py:TurnData.re_render()` triggers re-rendering when filters change

### Expand/Collapse Pattern (Existing)
The `TrackedContentBlock` renderer already implements a collapse/expand pattern:
- Collapsed: shows `"..."` placeholder
- Expanded: shows full content
- Arrow indicator: `\u25bc` (down arrow) when expanded, `\u25b6` (right arrow) when collapsed
- Controlled by `filters["expand"]` with per-block overrides via `expanded_overrides`

### Footer Bindings
Current bindings use single letters: h, t, s, e, m, a, c, l, f, j, k, n, N, g, G.
Available letters for new bindings: u, d, i, o, p, r, w, x, y, z, b, v, q.

## Gap Analysis

### What Needs to Be Built

1. **Two new filter states**: `show_user_messages` and `show_assistant_messages` (default: False = collapsed)
2. **Two new filter keys**: `"user"` and `"assistant"` in the filter system
3. **Modified rendering for TextContentBlock**: When preceded by a USER/ASSISTANT RoleBlock, collapse to first 2 lines with expand indicator
4. **Expand indicator**: Arrow shown only when message has >2 lines
5. **Key bindings**: Two new toggle keys
6. **Footer integration**: New bindings appear in footer with active state styling

### Design Decisions Required

1. **Where to implement collapse logic**: In the block renderer (`_render_text_content`) or at the `render_blocks` level?
   - Block renderers don't know their preceding blocks (no role context)
   - `render_blocks()` iterates sequentially and CAN track current role
   - **Decision**: Implement in `render_blocks()` which already handles context-dependent rendering (tool collapse)

2. **Filter semantics**: "collapsed by default, toggle to expand" means:
   - `show_user_messages = False` (default) = collapsed (first 2 lines)
   - `show_user_messages = True` = expanded (full content)
   - This matches existing patterns where `show_X = False` means reduced visibility

3. **Line counting**: How to count "2 lines" for collapsed view?
   - Count newlines in `TextContentBlock.text` before rendering
   - Use `text.splitlines()` and take first 2
