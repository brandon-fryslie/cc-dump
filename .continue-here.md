<decisions_made>

- **OpenAI Responses API, not Chat Completions**: Discovered from HAR that Copilot CLI uses `/responses` endpoint (OpenAI Responses API), NOT `/chat/completions`. Completely different SSE event types and request format.
- **upstream_format field instead of separate provider**: The proxy should not have a "copilot-translate" provider. Instead, `ProviderSpec.upstream_format` describes the upstream API shape. When it differs from `protocol_family`, translation activates. Dispatch tables keyed by upstream_format, not provider key.
- **--upstream copilot convenience flag**: Sets both target URL and upstream_format in one flag. Cleaner than two separate flags.
- **Auth from file**: Token read from `~/.local/share/copilot-api/github_token` (GitHub OAuth token, same as Copilot CLI uses)
- **Model mapping**: Hardcoded map of Anthropic model names → Copilot model names. Unknown models pass through.
- **Reasoning blocks deferred**: Copilot returns encrypted reasoning — skip for now, translate to thinking blocks later.
- **protocol_family stays as-is**: It describes the client-facing/display format. upstream_format is the new axis for the upstream API shape. They're independent.
</decisions_made>

<context>
The key architectural insight from this session: the proxy had been conflating "what format does the client speak" (protocol_family) with "what format does the upstream speak" (upstream_format). These were always the same until translation was needed. Adding upstream_format as an explicit, always-present field on ProviderSpec makes translation a natural consequence of the data, not a special mode.

The Copilot API uses `api.individual.githubcopilot.com` (not `api.githubcopilot.com` which is the older endpoint). Auth is a GitHub OAuth token (gho_...) from the file system. Required headers include Copilot-Integration-Id and X-GitHub-Api-Version.

The existing copilot provider in the registry (forward proxy mode, openai protocol) is for tools that already speak OpenAI Chat Completions natively — it's unrelated to the translation work.
</context>
